apiCamera( method, url )

La fonction qui représente la base de l'application. Elle prend en paramètre une méthode ainsi qu'une URL et retourne une nouvelle Promesse

ApiCamera retourne une nouvelle Promesse qui prendra en paramètre une méthode (de manière générale ce sera "GET") ainsi qu'une URL de l'API.

  • En cas de succés: celui-ci retournera la réponse émise par l'API.
  • En cas d'erreur: on récupère un code d'état.

Camera

Cette classe reprèsente la base de l'application. Elle prend dans son constructeur tous les champs que l'api nous retourne (id, description, image, lentilles, nom et le prix).

La class Camera créer un nouvel objet et prend comme valeurs :

  • L'id
  • La description
  • L'url d'une image
  • Un objet qui contient toutes les valeurs de lentilles pour caméra
  • Le nom de la caméra
  • Son prix

AddProduct

La classe AddProduct permet de créer de nouveaux objets de produits pour, plus tard, l'ajout dans le panier. Dans son constructeur, on y retrouve : l'id, le nom, l'image, le prix et sa quantité

La class AddProduct créer un nouvel objet pour l'ajout dans le panier. Il prend comme valeurs :

  • L'id
  • Le nom de la caméra
  • L'url de l'image
  • Son prix
  • Sa quantité

NewForm

NewForm est une classe qui créera de nouveaux objets pour l'envoie du formulaire. Celui-ci fera des vérifications au niveau des données qu'on lui passera dans son constructeur (Prénom, nom, email, N° de rue, adresse, code postal, ville, carte de crédit, date d'expiration et le code CVC). On y retrouve aussi deux champs qui n'ont pas besoin de vérification : l'id et le prix.

La class NewForm créer un nouvel objet qui prend en valeur :

  • Le prénom
  • Le nom
  • L'adresse email
  • Le numéro de rue
  • L'adresse
  • Le code postal
  • La ville
  • Le numéro de carte de crédit
  • La date d'expiration
  • Le code CVC
  • Le prix total
  • Un id

En plus de cette création d'objet, on y retrouve sept fonction qui testera les valeurs données avec du regex :

  • isValidCharacter : Vérifie si l'élément possède uniquement des lettres et pas des chiffres ou des caractères spéciaux.
  • isValidNumber : Vérifie si l'élément est bien un nombre
  • isValidCreditCard : Vérifie si l'élémentsest bien un numéro de carte de crédit et qui commence par le chiffre 4 ou 5 pour les numéro de carte masterCard et Visa
  • isValidDate : Vérifie si l'élément est bien une date (data qui serait au minimum en 2020)
  • isValidMail : Vérifie si l'élément est bien une adresse mail
  • isValidCVC : Vérifie si l'élément possède uniqument des chiffres (3 chiffres maximum).
  • isEmpty : Vérifie si l'élément n'est pas vide

badgeCount( )

J'incrémente ensuite "quantityCart" de la valeur de "add" de chaque produit existant dans "getJsonItems()" et je définit la syntaxe HTML de l'élément "badge" avec "quantityCart".

  • "noProducts" récupère l'élément html qui possède l'id : "not-products"
  • "priceDropdown" récupère l'élément html qui possède l'id : "price-dropdown"

Je vérifie si "getJsonItems()" n'est pas égale à "null".

Si c'est le cas ,J'initialise une variable "productCart" qui va créer un nouveau tableau avec les résultats de l'appel de la fonction "getJsonItems()" et je lui passe la méthode "join('')" afin de supprimer les ","

displayPrice( )

Cette fonction calcule la valeur total du prix de tous les produits existant dans le localStorage. Elle prend en compte la quantité d'un produit et le nombre total de produit. On y retrouve un aperçu dans le dropdown de la barre de navigation, la page panier et la page checkout.

Explication de la fonction displayPrice :

Pour commencer, j'initialise deux variable nommé "totalPrice" et "total". Dans la première variable, je récupère tous les éléments html "span" qui possèdent une class "total" avec "querySelector" et la seconde variable je lui met une valeur de "0".

Je vérifie si la longueur de "getJsonItems()" est supérieur à "0".

Si oui, je créer un nouveau tableau avec les résultats de l'appel de la fonction (je rajoute la méthode "join('')" afin de supprimer les virgules). J'incrément ensuite la variable "total" avec la fonction "price()" à qui je passe en paramètre le prix de chaque produit que je multiplie par sa quantité.

Je définit ensuite la syntaxe des élément html "totalPrice" par la valeur de "total".

Si la condition est fausse, je vérifie qu'un élément html avec un id "cart-empty" existe, et si c'est le cas, j'initialise deux variable ("badge" et "container") qui récupère les éléments html qui possède l'id "badge" et "container".

Je modifie la syntaxe de "container" et je rajoute une class "d-none" à "badge".

Plan de tests : Cart

Les éléments à vérifier :

  1. L'affichage des produits du panier

    D'abord on vérifie si il existe bien une ou plusieurs valeurs dans la clé Product du localStorage. Si c'est le cas, une boucle map( ) est faite sur ses éléments et nous retourne par la suite un block HTML qui affichera les lignes du tableau

  2. L'affichage du prix

    L'affichage du prix reprend le concept de l'affichage des produits en y ajoutant une condition sur la longueur des valeurs dans la clé Product du localStorage. Si celle-ci est supérieur à 0, elle calculera la valeur total du prix en prenant en compte la quantité de chaque produit. Sinon, elle écrase la syntaxe de la page panier pour afficher le message "Vous n'avez aucun produit dans votre panier. Retourner dans la boutique".

  3. L'incrémentation de la quantité d'un produit

    Si on souhaite augmenter la quantité d'un produit, addMoreProduct( ) récupère le produit en question dans un tableau secondaire qui reprend les valeurs du localStorage et incrémente la quantité du produit par 1 à chaque clique. Il met ensuite à jour le localStorage (updateLocalStorage( )) à l'aide du tableau secondaire et donne cet aspect dynamique au niveau du tableau de produit, du prix total, et du nombre de produits dans le panier.

  4. La décrémentation de la quantité d'un produit

    Si on souhaite diminuer la quantité d'un produit, removeMoreProduct( ) récupère le produit en question dans un tableau secondaire qui reprend les valeurs du localStorage et décrémente la quantité du produit par 1 à chaque clique. Il met ensuite à jour le localStorage (updateLocalStorage( )) à l'aide du tableau secondaire et donne cet aspect dynamique au niveau du tableau de produit, du prix total, et du nombre de produits dans le panier. Si le produit en question est décrémenté jusqu'au 0, alors celui-ci est automatique supprimer du tableau secondaire qui mettra ensuite à jour le localStorage et tous le processus qui en découle.

  5. La suppression d'un produit

    Si on souhaite supprimer un produit, removeProductLocalStorage( ) récupère le produit en question dans un tableau secondaire qui reprend les valeurs du localStorage et le supprime de son tableau au clique. Il met ensuite à jour le localStorage (updateLocalStorage( )) à l'aide du tableau secondaire et donne cet aspect dynamique au niveau du tableau de produit, du prix total, et du nombre de produits dans le panier.

Plan de tests : Product

  1. Récupérer l'id du produit

    Ici on récupère l'id qui se trouve dans l'url grâce à URLSearchParams( ). A l'aide d'une boucle for et des éléments retourné avec URLSearchParams( ), on récupère l'id.

  4. L'ajout au panier

    Un seconde then( ) est fait pour l'événement au "click" du boutton Ajouter au panier. Celui-ci vérifie si ce produit n'existe pas dans le localStorage. Si c'est le cas, il ajoute le produit dans le localStorage. Sinon il incrémente la valeur de quantité d'un +1.

  5. Pour une meilleur expérience utilisateur

    Les appels des fonctions badgeCount( ) et cartDropdown( ) vont permettre d'avoir un visuel en temps réel sur le panier ainsi que sa quantité de produits. Une autre condition prend en compte la longueur de la clé Product du localStorage et, si celui-ci est vide, il nous retourne un message "Vous n'avez aucun produit dans votre panier" et le badge sera invisible.

Plan de tests : Thanks

Les éléments à vérifier :

  1. Récupérer l'id de commande

    Ici on récupère l'id qui se trouve dans l'url grâce à URLSearchParams( ). A l'aide d'une boucle for et des éléments retourné avec URLSearchParams( ), on récupère l'id.

  2. Les conditions d'affichage

    Quatre affichages différents sont programmer en fonction de quatre cas :

    • Si il existe un élément dans la clé Order du localStorage et si l'id récupéré par la valeur de cette clé est égale à l'id récupéré dans l'url alors on affiche le message de remerciement pour la commande.

    • Sinon si il existe un élément dans la clé Order du localStorage et si l'id récupéré par la valeur de cette clé n'est pas égale à l'id récupéré dans l'url et que le booléan de l'id de l'url est vrai, alors on affiche le message suivant : Désolé mais ce numéro de commande ne vous appartient pas. Merci de retourner dans la boutique ou d'aller sur votre numéro de commande

    • Sinon si le booléan de l'id récupéré dans l'url est à false, alors on retourne le message suivant : Désolé, aucun numéro de commande est stipulé. Merci de retourner dans la boutique

    • Sinon on retourne le message suivant : Aucune commande n'a été faite, merci de retourner dans la boutique

  3. Suppression de la commande

    Au click sur n'importe quel lien de cette page, la clé Order est automatiquement supprimé.
