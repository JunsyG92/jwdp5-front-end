apiCamera( method, url )

La fonction qui représente la base de l'application. Elle prend en paramètre une méthode ainsi qu'une URL et retourne une nouvelle Promesse

ApiCamera retourne une nouvelle Promesse qui prendra en paramètre une méthode (de manière générale ce sera "GET") ainsi qu'une URL de l'API.

  • En cas de succés: celui-ci retournera la réponse émise par l'API.
  • En cas d'erreur: on récupère un code d'état.

sendForm( method, url, jsonBody, redirection )

L'envoie du formulaire passe par une fonction qui prend quatre paramètres : une méthode, une URL, un corp de body et une redirection. Elle retourne une nouvelle Promesse.

SendForm s'occupe de l'envoie du formulaire de la page checkout. La fonction prend 4 paramètres : la méthode (ici ce sera "POST", une URL, le corp des données à envoyer ainsi qu'une redirection. Elle retourne une nouvelle promesse.

Dans cette promesse , on y créer un objet "XMLHttpRequest" et on lui passe cinq méthode :

  • Open : Prend en paramètre la méthode, l'url et un booléan true indiquant qu'il faut traiter la requête en asynchrone.
  • setRequestHeader : pour éditer le header de la requête
  • onload : retourne en cas de succés une redirection vers la page de remerciements
  • onerror : affiche un message dans la console en cas d'erreur
  • send : envoie le corp de la requête en format JSON

Camera

Cette classe reprèsente la base de l'application. Elle prend dans son constructeur tous les champs que l'api nous retourne (id, description, image, lentilles, nom et le prix).

La class Camera créer un nouvel objet et prend comme valeurs :

La class AddProduct créer un nouvel objet pour l'ajout dans le panier. Il prend comme valeurs :

NewForm

NewForm est une classe qui créera de nouveaux objets pour l'envoie du formulaire. Celui-ci fera des vérifications au niveau des données qu'on lui passera dans son constructeur (Prénom, nom, email, N° de rue, adresse, code postal, ville, carte de crédit, date d'expiration et le code CVC). On y retrouve aussi deux champs qui n'ont pas besoin de vérification : l'id et le prix.

La class NewForm créer un nouvel objet qui prend en valeur :

  • Le prénom
  • Le nom
  • L'adresse email
  • Le numéro de rue
  • L'adresse
  • Le code postal
  • La ville
  • Le numéro de carte de crédit
  • La date d'expiration
  • Le code CVC
  • Le prix total
  • Un id

En plus de cette création d'objet, on y retrouve sept fonction qui testera les valeurs données avec du regex :

  • isValidCharacter : Vérifie si l'élément possède uniquement des lettres et pas des chiffres ou des caractères spéciaux.
  • isValidNumber : Vérifie si l'élément est bien un nombre
  • isValidCreditCard : Vérifie si l'élémentsest bien un numéro de carte de crédit et qui commence par le chiffre 4 ou 5 pour les numéro de carte masterCard et Visa
  • isValidDate : Vérifie si l'élément est bien une date (data qui serait au minimum en 2020)
  • isValidMail : Vérifie si l'élément est bien une adresse mail
  • isValidCVC : Vérifie si l'élément possède uniqument des chiffres (3 chiffres maximum).
  • isEmpty : Vérifie si l'élément n'est pas vide

addMoreProduct( )

addMoreProduct s'occupe de l'incrémentation du produit qu'on retrouve dans le localStorage. On y retrouve un aperçu de cette fonction dans la page du panier.

Explication de la fonction addMoreProduct :

J'initialise un array vide qui s'appelle "arrayIds". Je fais une condition qui vérifie si la longueur de "getJsonItems()" est supérieur à 0.

Si oui, je créer un nouveau tableau avec les résultats de l'appel de la fonction et je push chaque résultat dans l'array "arrayIds".

J'initialise "removeProduct" qui récupére l'élément chaque élément html qui à pour id "more-" + l'id du produit.

Au click sur ces éléments html, j'incrémente la valeur de "add" du produit et j'appelle la fonction "updateLocalStorage()"" auquel je passe en paramètre "arrayIds".

badgeCount( )

Cette fonction permet d'afficher le nombre de produits existant dans le panier (c'est à dire dans la clé "Product" du localStorage). On y retrouve un aperçu dans toutes les pages du site.

Explication de la fonction badgeCount :

D'abord je vérifie si l'élément html avec l'id "badge" existe bien.

Si oui, je l'initialise dans une variable "let".

Je vérifie ensuite si aucun élément existe avec la fonction "getJsonItems()".

Si oui, j'ajoute la class "d-none" à "badge"

Sinon, j'initalise une variable "quantityCart" avec comme valeur "0" puis je fais une boucle "for" ou je récupère l'index des éléments dans "getJsonItems()". J'incrémente ensuite "quantityCart" de la valeur de "add" de chaque produit existant dans "getJsonItems()" et je définit la syntaxe HTML de l'élément "badge" avec "quantityCart".

cartChekcout( )

cartCheckout permet d'afficher le contenu du panier à partir des valeurs existante dans la clé "Produit" du localStorage. On y retrouve un aperçu dans la page checkout

Explication de la fonction cartChekcout :

J'initialise quatre variable qui récupère des éléments html :

  • "listChekout" récupère l'élément html qui possède l'id : "listChekout"
  • "quantityProduct" récupère l'élément html qui possède l'id : "quantityProduct"
  • "delivery" récupère l'élément html qui possède l'id : "delivery"
  • "getTotal" récupère l'élément html qui possède l'id : "getTotal"

Je vérifie si "getJsonItems()" me retourne "true".

Si c'est le cas, j'initialise une variable "productNumber" qui à pour valeur "0"

J'initialise une seconde variable qui va créer un nouveau tableau avec les résultats de l'appel de la fonction "getJsonItems()" et je lui passe la méthode "join('')" afin de supprimer les ",".

J'incrément "productNumber" par rapport aux quantité de chaque produits

Pour l'affichage du mot "produit" en HTML, je vérifie si la longueur de "getJsonItems()" est de "1" et que la quantité du produit en question est aussi de "1". Si c'est le cas j'affiche le mot "produit" au singulier, sinon je l'affiche au pluriel.

Je retourne ensuite un block html qui affichera les produits en question dans la page checkout côté "Résumé".

J'initialise un prix de livraison ("deliveryPrice") à "250".

Je modifie la syntaxe de "listChekout" et "delivery" avec les valeurs de "checkoutProduct" et "deliveryPrice €".

Je vérifie que "getJsonItems()" existe bien pour éviter les message d'erreur en console et je modifie la syntaxe de "getTotal" avec la valeur de "total" + "deliveryPrice".

cartDropdown( )

cartDropdown permet d'afficher le contenu du panier dans le hover (dropdown) à partir des valeurs existante dans la clé "Produit" du localStorage. Celui-ci se trouve au niveau de la barre de navigation (l'icône panier). On y retrouve un aperçu dans la page d'accueil et la page produit du site.

Explication de la fonction cartDropdown :

J'initialise quatre variable qui récupère des éléments html :

  • "listCart" récupère l'élément html qui possède l'id : "listCart"
  • "table" récupère l'élément html qui possède l'id : "table-dropdown"
  • "noProducts" récupère l'élément html qui possède l'id : "not-products"
  • "priceDropdown" récupère l'élément html qui possède l'id : "price-dropdown"

Je vérifie si "getJsonItems()" n'est pas égale à "null".

Si c'est le cas ,J'initialise une variable "productCart" qui va créer un nouveau tableau avec les résultats de l'appel de la fonction "getJsonItems()" et je lui passe la méthode "join('')" afin de supprimer les ","

Je retourne un block HTML contenant les lignes du tableau qu'on retrouve dans le hover du panier avec les valeurs de chaque produits.

Je rajoute une class "d-none" à "noProducts" et je lui retire la class "d-block".

Je rajoute une class "d-block" à "table".

Je retire la class "d-none" à "priceDropdown" et je définit la syntaxe "productCart" à "listCart". J'appel ensuite la fonction "displayPrice()".

En dehors de la première condition, j'en initialise une nouvelle en vérifiant si la longueur de "getJsonItems()" est strictement égale à "0".

Si c'est le cas, j'ajoute la class "d-none" à "table" et "priceDropdown" et la class "d-block" à "noProducts". Je retire la class "d-block" à "table" et je définit une syntaxe "Vous n'avez aucun produit dans votre panier" à "noProducts"

getJsonItems( )

getJsonItems retroune un array JSON.parse des éléments se trouvant dans la clé "Product" du localStorage.

Explication de la fonction getJsonItems :

Je vérifie si une clé nommé "Product" existe dans le localStorage

Si c'est le cas, j'initalise une variable nommé "keyStorage" où je stock les valeurs de la clé "Product" du localStorage.

J'initialise une seconde variable nommé "products" qui prend en valeur "keyStorage" en format "JSON".

Je retourne ensuite "products".

Si la clé "Products" n'existe pas dans le localStorage, je retourne un array vide que j'ai initialisé dans une variable "products".

removeMoreProduct( )

removeMoreProduct s'occupe de la décrémentation du produit qu'on retrouve dans le localStorage. Lorsque le produit posséde la valeur de "0" dans sa quantité, celui-ci est supprimer de la clé "Produit". On y retrouve un aperçu de cette fonction dans la page du panier.

Explication de la fonction removeMoreProduct :

J'initialise une variable "arrayIds" qui possède un array vide.

Je vérifie ensuite si la longueur de la fonction "getJsonItems()" est supérieur à "0".

Si oui, je créer un nouveau tableau avec les résultats de l'appel de la fonction et je push chaque résultat dans l'array "arrayIds"

J'initialise "removeProduct" qui récupére l'élément chaque élément html qui à pour id "less-" + l'id du produit.

Au click sur ces éléments html, je décrémente la valeur de "add" du produit et j'appelle la fonction "updateLocalStorage()"" auquel je passe en paramètre "arrayIds".

Pour mettre fin à cette décrémentation, je vérifie si la partie "add" du produit est à "0".

Si c'est le cas, j'initialise une variable nommé "findObject" je récupère le premier élément trouvé qui possède le même id que l'id du produit

Je récupère ensuite son index dans le tableau que j'initialise dans une variable nommé "keyObject".

displayPrice( )

Cette fonction calcule la valeur total du prix de tous les produits existant dans le localStorage. Elle prend en compte la quantité d'un produit et le nombre total de produit. On y retrouve un aperçu dans le dropdown de la barre de navigation, la page panier et la page checkout.

Explication de la fonction displayPrice :

Pour commencer, j'initialise deux variable nommé "totalPrice" et "total". Dans la première variable, je récupère tous les éléments html "span" qui possèdent une class "total" avec "querySelector" et la seconde variable je lui met une valeur de "0".

Je vérifie si la longueur de "getJsonItems()" est supérieur à "0".

Si oui, je créer un nouveau tableau avec les résultats de l'appel de la fonction (je rajoute la méthode "join('')" afin de supprimer les virgules). J'incrément ensuite la variable "total" avec la fonction "price()" à qui je passe en paramètre le prix de chaque produit que je multiplie par sa quantité.

Je définit ensuite la syntaxe des élément html "totalPrice" par la valeur de "total".

Si la condition est fausse, je vérifie qu'un élément html avec un id "cart-empty" existe, et si c'est le cas, j'initialise deux variable ("badge" et "container") qui récupère les éléments html qui possède l'id "badge" et "container".

Je modifie la syntaxe de "container" et je rajoute une class "d-none" à "badge".

updateLocalStorage( data )

Cette fonction permet de mettre à jour le localStorage après modification (par exemple, si on incrémente un produit à partir de la page panier, le localStorage, le prix totale et le nombre de produit dans le panier se met à jour).

Explication de la fonction updateLocalStorage :

La fonction prend un paramètre nommé : "data".

Je met à jour dans le localStorage la clé nommé "Product" et prend comme valeur le paramètre "data" convertit en chaîne JSON.

Je fais appel à la fonction "listProductCartFunction()" et "badgeCount()" pour mettre à jour la liste de produit du panier et le nombre de produit dans le badge du panier.
